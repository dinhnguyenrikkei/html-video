# Ngày 02: Sự khác biệt "một trời một vực" giữa code Phần mềm và code Phần cứng.

**Chủ đề (Topic):** Phân tích sự khác nhau giữa Software và Embedded.
**Phong cách hình ảnh (Visual Style):** Tech/Cyberpunk, chia đôi màn hình (Split screen), tông xanh dương (Software) vs tông cam/đỏ (Hardware).
**Giọng đọc (Voiceover):** Giọng nữ AI tiếng Việt, dứt khoát, chuyên môn.
**Nhạc nền (BGM):** Nhạc Synthwave dồn dập, nhịp điệu tương phản.

## Chi tiết Prompt từng cảnh (Scene-by-Scene)

**Cảnh 1 (0-5s): Hook**
- **Visual:** Chia đôi màn hình: Bên trái là một người gõ code trên IDE đen ngòm rồi chạy ra giao diện Web. Bên phải là gõ code rồi nạp thẳng vào một bảng mạch nhấp nháy đèn LED.
- **Voiceover:** Code phần mềm và code phần cứng - Khác nhau một trời một vực như thế nào?

**Cảnh 2 (5-15s): Phân tích**
- **Visual:** Bên trái (Software): Hiển thị icon RAM 16GB, CPU Core i7 (cảm giác dư thừa tài nguyên). Bên phải (Embedded): Hiển thị chip ARM, RAM chỉ vài Kilobytes, chớp nháy cảnh báo giới hạn tài nguyên.
- **Voiceover:** Code Web/App, bạn có RAM hàng Gigabyte để vung vẩy. Nhưng code Nhúng, bạn chỉ có vài Kilobyte tài nguyên. Viết một dòng code dư thừa là hệ thống sập toàn tập!

**Cảnh 3 (15-30s): Điểm nhấn**
- **Visual:** Cảnh zoom cận vào một kỹ sư đang dùng máy hiện sóng (Oscilloscope) để dò tín hiệu điện.
- **Voiceover:** Code phần mềm lỗi, bạn F5 là xong. Code Nhúng lỗi? Bạn phải cầm đồng hồ đo điện, đo từng chân chip để xem là lỗi code hay... cháy mạch! Đổi lại, cảm giác điều khiển vạn vật vật lý bằng chính những dòng code của mình là thứ phần mềm không bao giờ mang lại được.

**Cảnh 4 (30-40s): Call to Action**
- **Visual:** Logo HUST x Rikkei Academy. Text: "Chinh phục phần cứng cùng chuyên gia Bách Khoa".
- **Voiceover:** Chinh phục phần cứng không khó nếu có người dẫn đường. Bắt đầu lộ trình Lập trình Nhúng thực chiến cùng chuyên gia Bách Khoa ngay hôm nay. Link ở dưới bình luận!
